# CI 修复总结 — commit `9146f24`

## 日期：2026-07-24

## 问题

commit `44fded6` 推送后 CI 连续失败（3 次），原因：

### 错误 1：`PlayerViewModel.kt` — `isVlcEngine` 冲突声明
- 第 91 行：`val isVlcEngine: StateFlow<Boolean> = _isVlcEngine.asStateFlow()`
- 第 99 行：`val isVlcEngine: Boolean get() = engineType.value == EngineType.VLC`
- 两个同名属性 `isVlcEngine` 类型不同，Kotlin 编译器报 `Conflicting declarations`

### 错误 2：`PlayerScreen.kt:120` — `Overload resolution ambiguity`
- `playerVm.isVlcEngine` 有两个声明（StateFlow 和 Boolean），`collectAsStateWithLifecycle()` 不知道调用哪个重载
- 根因同样是 `isVlcEngine` 冲突声明

### 错误 3（首次修复尝试）：`DisposableEffect` 返回类型
- `DisposableEffect(isFullScreen)` 的 `if` 分支不满足时没有 `onDispose`，导致 `DisposableEffectResult` 缺失
- 修复：添加 `else { onDispose {} }`

## 修复

1. **删除 `PlayerViewModel.kt` 第 99 行**：移除冗余的 `val isVlcEngine: Boolean get() = ...`，保留 `StateFlow<Boolean>` 版本
2. **`PlayerScreen.kt` DisposableEffect**：添加 `else { onDispose {} }` 分支确保所有路径返回 `DisposableEffectResult`
